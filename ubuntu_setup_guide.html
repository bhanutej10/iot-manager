Install Docker on Ubuntu
Official Docker Engine — not snap, not ubuntu default
Use Docker's official apt repositoryUbuntu's default apt install docker.io installs an older version. Follow these exact steps from Docker's official repository to get Docker Engine with Compose v2 support.
🔑Step 2.1 — Remove old Docker, add GPG key and repositoryOfficial source
bash — copy and run all at once
# Remove old Docker installations
sudo apt remove -y docker docker-engine docker.io containerd runc 2>/dev/null || true

# Add Docker's official GPG key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
    sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# Add Docker stable repository
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Install Docker Engine + Compose v2 plugin
sudo apt update
sudo apt install -y \
    docker-ce docker-ce-cli containerd.io \
    docker-buildx-plugin docker-compose-plugin
👤Step 2.2 — Add your user to docker groupCritical Ubuntu step

By default on Ubuntu, only root can run Docker. Add yourself to the docker group so you don't need sudo for every command.

bash
sudo usermod -aG docker $USER
newgrp docker          # apply without logout — OR log out and back in
sudo systemctl enable docker
sudo systemctl start docker

Troubleshooting — Ubuntu Fixes
Solutions to the most common Ubuntu-specific problems
🚫Permission denied connecting to Docker daemon
fix
sudo usermod -aG docker $USER && newgrp docker
docker ps   # must work without sudo
🚫Port 1883 or 80 already in use
fix
sudo systemctl stop mosquitto apache2 nginx 2>/dev/null || true
sudo lsof -i :1883 -i :80 -i :5000   # check what's still running
docker compose up --build
🚫pip install scapy fails — missing libpcap
fix
sudo apt install -y libpcap-dev gcc python3.11-dev
pip install --force-reinstall scapy==2.5.0
🚫No sensor data — readings_last_hour: 0
fix
docker compose logs simulator | tail -10
# [10:23:42] Tick #5 — 10 sensors — 4821 bytes  ← must appear
docker compose restart simulator backend
🚫"docker compose" command not found
fix
sudo apt install -y docker-compose-plugin
docker compose version   # must show v2.x.x
🔄Complete reset — wipe everything and rebuild
bash
cd ~/iot-manager
docker compose down -v           # stop + wipe volumes (database)
docker compose down --rmi local  # remove built images
docker system prune -f           # free disk space
docker compose up --build        # clean rebuild from scratch
